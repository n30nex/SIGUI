# Known Limitations

As of the 2026-06-29 D1L hardware validation:

- Manual visual confirmation of display bars and touch target movement is still pending; the serial `display test` and `touch test` commands return OK on hardware.
- MeshCore Public group text TX/RX and signed advert TX/RX are implemented and validated through the serial console, firmware-local contacts can promote heard nodes by fingerprint, full advertised public keys are retained for DM targeting and have passed reboot persistence on `COM7`, contact favorite/mute flags persist across reboot, Public unread/read state persists across reboot, firmware-local routes are learned from Public/advert path metadata, route detail diagnostics and a first route detail sheet are implemented, and serial DM flood TX plus the bounded DM store are validated. DM ACK/PATH receive parsing, direct-route TX selection, contact-row DM compose, first contact detail actions, and a first bounded DM thread/detail sheet are implemented, but manual touch review, controlled ACK/PATH RF proof, inbound DM proof, controlled inbound DM unread proof, and direct-route RF proof are still pending.
- Settings are implemented as an NVS-backed firmware model and have passed reboot persistence smoke on real D1L flash. The persisted radio profile is now exposed through `settings get`, serial radio setters, and a staged touch Radio Settings sheet; the radio editor slice has passed hardware flash, standard smoke, targeted serial profile proof, and active Public `test` RF regression on `COM7`. Manual touch review is still pending.
- First-boot onboarding now persists a setup-complete flag, collects the node name, confirms Canada/USA defaults, keeps Wi-Fi/BLE/observer disabled, and generates/retains the local MeshCore identity. Optional-radio choices in onboarding remain limited until live Wi-Fi/BLE runtime support is enabled.
- `identity status` now generates and reports an NVS-retained Ed25519 local identity fingerprint. This is a minimal firmware identity model, not the final full MeshCore C++ store integration.
- MeshCore TX timestamps are retained monotonically in NVS to avoid duplicate filtering, but a real RTC/network time source is not wired yet.
- MeshCore 3-byte companion framing is implemented and host-tested, but the live binary bridge to MeshCore protocol frames is not enabled in Phase 1.
- Packet log storage keeps a 32-row RAM ring and persists the newest 8 rows in NVS. Packet rows now carry a bounded raw hex preview and expose serial filter/search/raw commands plus touch Packet-tab filters/search; larger retention, export, and deeper protocol decode are still pending.
- Phase 6 mesh visibility now exposes read-only `signal`, `roomservers`, and `repeaters` diagnostics plus Home/Packet-tab summary cards, and the serial/summary slice is built, flashed, smoke-tested, reboot-checked, and Public `test` RF-regression tested on `COM7`. A first touch Mesh Roles browser sheet is also built, flashed, smoke-tested, reboot-checked, and RF-regression tested on `COM7`; manual physical touch confirmation of opening/scrolling/closing it is still pending. Trace/ping helpers and richer filters are still pending.
- Contact export now emits MeshCore-compatible `meshcore://contact/add?...` payloads over serial and adds a Contact Detail `Export` action with LVGL QR rendering enabled in the default build. It requires a promoted contact with a retained 64-hex public key. The QR-enabled build passed COM7 flash, standard smoke, targeted `contacts export <fingerprint>` proof, and active Public `test` RF regression; manual touch open/close review and scan/import proof with a MeshCore client are still pending.
- The Phase 3/4/5/6 touch shell is implemented with first-boot onboarding, Home, Public `test` action, first Public free-text composer, first contact-row DM composer, first contact detail sheet with DM/favorite/mute actions, first bounded DM thread/detail sheet with `Reply`, persisted recent Public/DM preview rows, Public/DM unread counters, newest contact/heard-node rows, Home/Packet-tab signal and mesh-role summary cards, first Mesh Roles browser sheet, Packet-tab route rows with a first route detail sheet, Packet-tab packet rows with filters/search/raw-hex detail, Settings companion status, a staged Radio Settings sheet, advert sheet, toasts, and lock overlay, but full Public scrollback/search, richer full-thread DM history, per-thread DM read cursors, richer contact editing, route trace/ping helpers, BLE companion transport, live Wi-Fi scan/connect, and OTA are not implemented yet.
- `tools/ui_simulator.py` now provides deterministic 480x480 host screenshots, required-label/text-overflow checks, and a `large-mesh` stress scenario for oversized node/message stores. This is a fast CI regression check, not a replacement for the pending manual physical-screen review of touch behavior, LCD color/brightness, animation smoothness, or genuinely large live RF mesh behavior.
- The Public composer, contact-row DM composer, contact detail sheet, contact export sheet, Radio Settings sheet, bounded DM thread sheet, Messages unread/read action, route detail sheet, and packet detail sheet have passed host contract/simulator tests. The contact export and radio settings additions have also passed hardware flash, standard smoke, targeted serial proof, and active Public `test` RF regression on `COM7`; manual physical touch entry on the composer keyboard and thread/reply/contact/export/radio/read/route-detail/packet-detail surfaces is still pending.
- The SX1262 Phase 1 probe intentionally avoids TCXO/DIO3 control. TCXO remains `NONE` by default.
- Wi-Fi and BLE are managed by a Phase 5 status/persistence foundation. This build reports Wi-Fi support compiled in but not started by DeskOS, and BLE build-disabled, so `wifi scan` reports why scanning did not start instead of listing networks.
- Phase 7 diagnostics now expose a small reset/crash ring and richer health telemetry. A repeatable soak runner now samples health, mesh, signal, unread, packet, command retries, and crashlog state for idle/listening or active Public `test` windows, and a retry-aware 1-hour active Public `test` hardware soak passed on `COM7`; the full 12-hour idle/listening soak is still pending.
