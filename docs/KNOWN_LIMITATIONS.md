# Known Limitations

As of the 2026-06-29 D1L hardware validation:

- Manual visual confirmation of display bars and touch target movement is still pending; the serial `display test` and `touch test` commands return OK on hardware.
- MeshCore Public group text TX/RX and signed advert TX/RX are implemented and validated through the serial console, firmware-local contacts can promote heard nodes by fingerprint, full advertised public keys are retained for DM targeting and have passed reboot persistence on `COM7`, contact favorite/mute flags persist across reboot, Public unread/read state persists across reboot, firmware-local routes are learned from Public/advert path metadata, route detail diagnostics and a first route detail sheet are implemented, and serial DM flood TX plus the bounded DM store are validated. DM ACK/PATH receive parsing, direct-route TX selection, contact-row DM compose, first contact detail actions, and a first bounded DM thread/detail sheet are implemented, but manual touch review, controlled ACK/PATH RF proof, inbound DM proof, controlled inbound DM unread proof, and direct-route RF proof are still pending.
- Settings are implemented as an NVS-backed firmware model and have passed reboot persistence smoke on real D1L flash.
- `identity status` now generates and reports an NVS-retained Ed25519 local identity fingerprint. This is a minimal firmware identity model, not the final full MeshCore C++ store integration.
- MeshCore TX timestamps are retained monotonically in NVS to avoid duplicate filtering, but a real RTC/network time source is not wired yet.
- MeshCore 3-byte companion framing is implemented and host-tested, but the live binary bridge to MeshCore protocol frames is not enabled in Phase 1.
- Packet log storage keeps a 32-row RAM ring and persists the newest 8 rows in NVS. Richer packet filtering/search and raw packet hex developer mode are still pending.
- Phase 6 mesh visibility now exposes read-only `signal`, `roomservers`, and `repeaters` diagnostics plus Home/Packet-tab summary cards, and the slice is built, flashed, smoke-tested, reboot-checked, and Public `test` RF-regression tested on `COM7`. Dedicated full-screen repeater and room-server browsers, trace/ping helpers, and richer filters are still pending.
- The Phase 3/4/5/6 touch shell is implemented with Home, Public `test` action, first Public free-text composer, first contact-row DM composer, first contact detail sheet with DM/favorite/mute actions, first bounded DM thread/detail sheet with `Reply`, persisted recent Public/DM preview rows, Public/DM unread counters, newest contact/heard-node rows, Home/Packet-tab signal and mesh-role summary cards, Packet-tab route rows with a first route detail sheet, Packet-tab packet rows with a first packet detail sheet, Settings companion status, advert sheet, toasts, and lock overlay, but full Public scrollback/search, richer full-thread DM history, per-thread DM read cursors, richer contact editing, route trace/ping helpers, dedicated room-server/repeater list screens, packet filtering/search, BLE companion transport, live Wi-Fi scan/connect, and OTA are not implemented yet.
- The Public composer, contact-row DM composer, contact detail sheet, bounded DM thread sheet, Messages unread/read action, route detail sheet, and packet detail sheet have passed host contract tests, local ESP-IDF build, flash, and serial smoke/backend probes; manual physical touch entry on the composer keyboard and thread/reply/contact/read/route-detail/packet-detail surfaces is still pending.
- The SX1262 Phase 1 probe intentionally avoids TCXO/DIO3 control. TCXO remains `NONE` by default.
- Wi-Fi and BLE are managed by a Phase 5 status/persistence foundation. This build reports Wi-Fi support compiled in but not started by DeskOS, and BLE build-disabled, so `wifi scan` reports why scanning did not start instead of listing networks.
- Phase 7 diagnostics now expose a small reset/crash ring and richer health telemetry, but the 12-hour idle/listening soak and 1-hour active messaging soak are still pending.
